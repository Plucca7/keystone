# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in a project generated from this
template, please report it privately instead of opening a public issue —
a public report gives attackers a head start before a fix ships.

Report privately through one of these channels:

- **GitHub private vulnerability reporting** (preferred): open the
  repository's "Security" tab and select "Report a vulnerability". This
  creates a private advisory visible only to maintainers until a fix is
  ready.
- **Email**: `security@example.com` (replace this placeholder with a real
  address you monitor before publishing your project).

Please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce it (a minimal example is ideal).
- The affected version or commit.

## What to expect

- Acknowledgement of your report within a reasonable timeframe.
- An assessment of severity and, where confirmed, a plan and rough timeline
  for a fix.
- Credit in the fix's release notes, if you would like it and once the
  vulnerability is resolved.

## Scope

This policy covers the code in this repository. It does not cover
third-party dependencies — report those directly to their maintainers, or
through the ecosystem's advisory database (for example npm's
`advisories`).
